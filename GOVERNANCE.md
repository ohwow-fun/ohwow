# Governance

ohwow is maintained by its creator (@ogsus) who serves as the project's Benevolent Dictator For Life (BDFL).

## Decision Making

- Minor changes (bug fixes, docs, agent templates): merged by any maintainer
- Feature additions: require BDFL approval
- Breaking changes: announced in advance, discussed in GitHub Discussions or Discord
- Architecture changes: require an RFC (issue with the `rfc` label)

## Becoming a Maintainer

Regular contributors who demonstrate consistent quality contributions may be invited to become maintainers. Maintainers can merge PRs and triage issues.

## Response Times

We aim to:
- Acknowledge new issues within 48 hours
- Provide initial PR review within 72 hours
- Respond to security reports within 48 hours (see SECURITY.md)

## Communication

- GitHub Issues: bugs, features, RFCs
- GitHub Discussions: questions, ideas, showcases
- [Discord](https://discord.gg/WUGnGqceeY): real-time chat
